Access to people.uic.edu and webhost.uic.edu

  • SSH shell access
    • PuTTY from windows
    • Terminal access with Mac/Linux
  • SFTP
  • SMB/CIFS
    • people.uic.edu
      • Windows UNC path: \\people.uic.edu\<NETID>
      • From Mac or Linux: smb://people.uic.edu/<NETID>
      • Document root in public_html directory
    • webhost.uic.edu
      • Windows UNC path: \\webhost.uic.edu\<SITENAME>
      • From Mac or Linux: smb://webhost.uic.edu/<SITENAME>
      • Document root in htdocs directory
  • VPN required for Access off campus (openvpn.uic.edu)

File Permissions

  • Unix file permissions
    • Three scopes checked in order: User, Group, Others
    • Three types of access
      • r - read access
      • w - write access
      • x - execute access on a file, content access on a directory
  • people.uic.edu - Only owner and apache have access.
  • webhost.uic.edu - Only content managers and apache have access.

PHP Web development

  • Simply upload your PHP to your document root
    • No more ugly codewrap URLs!
    • PHP 5.3.3
    • PHP must be enabled on webhost
  • Start new with best practices:
    • Don't hard code configuration
    • Separate your config and data from your app
    • Let us handle your authentication
    • Design patterns - MVC, REST
    • Version control with git

How to avoid hard-coded configuration

  • Example in no-hardcode/
  • Read values dynamically:
    $hostname = $_SERVER["HTTP_HOST"];
$page_url = $_SERVER["SCRIPT_URI"];
$php_user_id = posix_geteuid();
$php_user_info = posix_getpwuid ( $php_user_id );
$sitehome = $php_user_info['dir'];
$phpdir = dirname(__FILE__);

Using configs with PHP is easy!

  • Example in read-config/
  • Just call parse_ini_file()
  • Server will automatically block access to .ini files in document root
  • Still, it's best to set permissions to block other and apache access
  • Better yet, keep you config files out of your document root

PHP Authentication Tips

  • Let us handle the logins.
  • To force login, just used an allowed.netids file:
    • Example in login-required/
    • Comment your allowed.netids using #
    • To allow a specific list of users:
      # Fictional characters permitted
hpotter
dvader
    • To allow everyone at UIC:
      <allow groups>
all
  • For optional login, redirect users to /bluestem/cgi/lb_login.cgi/path/to/app
    • Example in login-optional/
  • In your PHP, check $_SERVER[REMOTE_USER] and $_SERVER[REDIRECT_REMOTE_USER] (webhost)

Using LDAP for user information

  • LDAP for user identification
    • Example in login-userinfo/
    • Server: ldap.uic.edu
    • Base DN: ou=people,dc=uic,dc=edu
    • Search: (&(uid=$this->netid)(primarypid=true))
    • Interesting Attributes: displayname, ou
